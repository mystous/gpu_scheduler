# 페르소나 리뷰 원칙

제출 전, 5인 가상 심사단([persona/](persona/))으로 **현재 논문을 모의 심사**하고 그 결과를 누적 기록한다.

## 원칙
1. **대상은 항상 현재 상태**: 리뷰 시점의 `paper/sn-article.tex`(+그림·표)와 그 commit을 명시한다. 옛 버전이 아니라 지금 빌드되는 논문을 본다.
2. **5인 전원이 자기 렌즈로**: 각 페르소나는 자기 [전문/편향/평가 성향]에 충실하게, 그리고 그 페르소나의 [실제 리젝 이력 근거]를 반영해 리뷰한다. 호의적으로 왜곡하지 않는다 — *진짜 리뷰어처럼 가장 아픈 곳을 찌른다*.
3. **항목 고정**: 각 리뷰는 `점수(IEEE 척도) / 요약 / 강점 / 약점·우려 / 질문 / 권고`. 점수는 Accept(2)·Weak Accept(1)·Borderline(0)·Weak Reject(−1)·Reject(−2)·Strong Reject(−3).
4. **합의 블로커 추출**: 끝에 *3인 이상이 동시에 제기한 사유*를 모아 **제출 차단 항목**으로 둔다. 이게 다음 작업의 최우선이다.
5. **이력 추적**: 직전 리뷰의 블로커가 해소됐는지, 새 블로커가 생겼는지 비교한다.
6. **정직성 우선**: 과장·미해결을 숨기지 않는다. 모의 리뷰가 관대하면 실제 심사에서 터진다.

## 산출물
- 회차별 파일: `review_prep/review_history/{YYYY-MM-DD_HHMMSS}.md` (이 디렉토리에 누적).
- 파일 머리에 대상 commit·제목·한 줄 총평. 본문에 5인 리뷰 + 합의 블로커 + 액션 아이템.

## 사용
- 모든 합의 블로커가 닫히기 전엔 제출하지 않는다(gate).
- 막히는 "예상 질문" = 보강 지점. 답할 수 있으면 통과.
